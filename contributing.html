Contributing

Contribute to DAQExpert step by step guide 

$> git clone https://github.com/cmsdaq/DAQExpert.git
$> git checkout experimental

Step 1 Get DAQExpert from Github

Checkout project source into your favorite IDE. Switch to experimental branch. In this guide Eclipse is used.

Download Eclipse Go to repo

Step 2 Import to IDE

In Eclipse import project as Maven project: FileImportExisting Maven Projects

Step 3 Get Tomcat

DaqExpert is a web application. Download the latest Tomcat servlet container (Tomcat 8 recomended).

Get Apache Tomcat 8

Step 4 Set up Tomcat in Eclipse

Right click inside "Server" tabNewServerTomcatChoose the directory with your Tomcat download. 

$> ssh cmsusr \
"tar -c -C /daqexpert/snapshots/pro/ ./2016/10/5/8/" \
| tar -x -C /your-snapshots-destination/

Step 5 Get test snapshots

DAQ snapshots are localized in /daqexpert/snapshots/pro/ in P5 cluster.

They are organized in a time-based directory structure (e.g. 2016/9/31/21 - data for 2016-09-31 21:00 UTC, local time 2016-09-31 23:00 CEST) 

$> cd /your-project-dir/src/main/resources/
$> vim config.properties
snapshots=/your-snapshots-destination/

Step 6 Configure

There is example configuration file config.properties . Specify your snapshots directory with snapshots property.

Step 7 Contribute

Write your Logic Module (LM) . You can start by duplicating the sketch SimpleSketch*

Please keep in mind:

  • Don't repeat yourself - reuse** results of LMs for better performance and maintainability
  • One to one - focus on one apect/problem in single LM, avoid multi-purpose LMs

* located in rcms.utilities.daqexpert.reasoning.logic.experimental package

** find many useful LMs in rcms.utilities.daqexpert.reasoning.logic.basic package, (e.g. NoRate, RunOngoing, Downtime)
