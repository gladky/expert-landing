Contribute to DAQExpert step by step guide

What is Logic Module?

Logic Module (abbr. LM) is a building block of DAQExpert expert logic.

  • Each LM defines one condition
  • The definition of condition is placed in satisfy method
  • If the method returns true if condition is satisfied and false otherwise
  • One LM can use results of another LM

What can I do in LM?

Logic Module allows you to focus on one thing: expressing your knowledge about DAQ system.

  • Easily access DAQ snapshot
  • Perform your checks on DAQ snapshot

Just return true when your condition is satisfied and the framework will do the rest (visualize, generate notifications, persist results)

$> ssh -N cmsusr

Step 1 Set up the tunnel

DAQExpert is available only in CMS cluster network.

Step 2 Run some sketch LMs

To run experimental Logic Module:

  1. Switch to Experimental mode in browser view
  2. Click Run Experimental
  3. Choose LM from dropdown menu
  4. Click Run
Results of your Experimental LM and other LMs will be updated.

Note that you can compare the results by switching between Experimental and Extended modes.

$> cd /daqexpert/experimental/pro/
$> cp SketchLogicModule.java MyLogicModule.java

Step 3 Make your own experimental LM

The best way to start is to copy the empty sketch LM.

Your file will be immediately available to run in the dropdown list from last step. No compilation required!

Step 4 Edit your logic module

Now use your favorite editor to make this Logic Module useful.

Express your expert knowledge by following the TODO comments from sketch file.

  1. Set up basic information
  2. Implement your condition

Step 5 Verify the results

Run your LM as you did in step 2 (except choose your LM from dropdown menu)
